Upload Class CSV Files

Upload one or more class CSV files for a single term, into the active academic session. Class name is detected from each file name (e.g. Class6.csv → "Class6") unless you override it below.

Automatic data cleaning rules applied on import:
• Any column header named SST is mapped directly to the Social subject field.
• Name variants such as "Ejaz nawaz" and "Ajaz Nawaz" are consolidated into one canonical student record to avoid duplicates.
• If a GR No column is present, it is used as the permanent match key (recommended for multi-session continuity); otherwise matching falls back to normalized name within the same class.
• A Section column, if present, is captured alongside Class.
Importing into active session: — switch sessions from the top bar or Settings if needed.
Link a Session Folder — Local Computer (No Re-selecting Files Every Time)

Link a local folder — e.g. a 2025-26 folder containing KG A.csv, KG B.csv, Nursery A.csv — once, and load every class CSV inside it with one click each time you open the app. Requires Chrome or Edge on desktop. Works fully offline.

Import From Public Links (Simplest — No Google Cloud Setup, Fully Free)

Note: Google Sheets "publish to web" links don't work here — Google's servers never send the permission headers browsers require, no matter how the link is formatted, so those will always fail. What genuinely works for free with no sign-in: upload your class CSVs to a free public GitHub repository, click the file, click the "Raw" button, and copy that address (it looks like https://raw.githubusercontent.com/yourname/yourrepo/main/ClassA.csv). Paste one such link per line below.

Works from any device with internet, completely free, no account setup beyond a free GitHub account. The trade-off: the repository must be public — anyone with the link (or who finds the repo) can view the files. If that's not acceptable for your data, use the Google Drive OAuth option below instead, which keeps files private.
Link a Session Folder — Google Drive OAuth (Access a Private Folder)

If you'd rather link an actual private Drive folder (not a publicly-shared file) and have every CSV inside it load automatically, this needs a one-time Google Cloud OAuth Client ID — free to create, but more setup than the public-link method above.

Needs a Google Cloud project with the Drive API enabled and an OAuth Client ID (type "Web application") with this page's address added as an authorized JavaScript origin. This page must be hosted at a real http(s) address — opening the file directly (file://) won't allow Google sign-in to complete.

Subject Total Marks

Every subject defaults to 100 total marks. Since some classes mark the same subject out of a different total (e.g. English out of 50 in Play Group but 100 in upper classes), pick a scope below: set the Global Default for the general fallback, or pick a specific class to override just that class's total. Class overrides always win over the global default. New subjects are added automatically the first time they appear in an uploaded CSV.

Document Titles

Academic Sessions

Every student record belongs to a session (e.g. 2026-2027). Create a new session each academic year — optionally carrying the roster forward — while every past session's results stay permanently attached and viewable.

How session-linking works: each student keeps a permanent GR No. When you upload a new session's CSVs with the same GR No, that student's new term marks attach to the same identity, and their full history stays browsable via "View Multi-Session History" in the Report Engine. Always include a GR No column in your CSVs for reliable continuity across years.

Cloud Sync Settings — Supabase

Connect this portal to a Supabase project for live cross-device sync of the student register and branding.

Required tables (run once in the Supabase SQL editor):
create table sms_students (id int primary key, data text, updated_at timestamptz);
create table sms_branding (id int primary key, data text, updated_at timestamptz);
Both tables need Row Level Security disabled or an appropriate anon-access policy for this prototype to read/write. The students payload includes every session, so pulling on another device restores full history.
